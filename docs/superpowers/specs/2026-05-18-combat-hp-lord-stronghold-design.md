# 战斗血量系统 + 领主即移动据点 设计规格

## 概述

将现有的石头剪刀布一击必杀战斗系统改为**数值制血量+伤害系统**，保留兵种克制关系（体现为伤害倍率）。同时将固定据点与领主合并，领主本身即为**可移动据点**，支持攻击/防守双模式切换。

---

## 一、士兵血量与伤害系统

### 1.1 兵种数值表

| 兵种 | HP | ATK | 攻击间隔(秒) | 攻击方式 | 造价(不变) |
|------|-----|-----|-------------|---------|-----------|
| 农民 | 20 | 5 | 1.0 | 近战 | 10木材 |
| 士兵 | 60 | 25 | 0.8 | 近战 | 20木材(+1农民转化) |
| 骑士 | 120 | 35 | 1.2 | 近战 | 15石头(+1农民转化) |
| 弓手 | 40 | 20 | 1.5 | 远程(射程150像素) | 10木+10石(+1农民转化) |

Config 新增字段：
```lua
UnitStats = {
    peasant  = { hp = 20,  atk = 5,  atkInterval = 1.0 },
    soldier  = { hp = 60,  atk = 25, atkInterval = 0.8 },
    knight   = { hp = 120, atk = 35, atkInterval = 1.2 },
    archer   = { hp = 40,  atk = 20, atkInterval = 1.5 },
},
```

### 1.2 克制倍率表

实际伤害 = ATK × 倍率。血月期间再 ×1.5。

| 攻击方 ↓ \ 防御方 → | 士兵 | 骑士 | 弓手 | 农民 |
|---------------------|------|------|------|------|
| **士兵** | 1.0 | 0.6 | 1.5 | 2.0 |
| **骑士** | 1.3 | 1.0 | 1.0 | 2.0 |
| **弓手** | 0.7 | 1.3 | 1.0 | 2.0 |
| **农民** | 0.5 | 0.3 | 0.5 | 1.0 |

Config 新增字段：
```lua
DamageMultiplier = {
    soldier = { soldier = 1.0, knight = 0.6, archer = 1.5, peasant = 2.0 },
    knight  = { soldier = 1.3, knight = 1.0, archer = 1.0, peasant = 2.0 },
    archer  = { soldier = 0.7, knight = 1.3, archer = 1.0, peasant = 2.0 },
    peasant = { soldier = 0.5, knight = 0.3, archer = 0.5, peasant = 1.0 },
},
```

### 1.3 Follower 实体字段变化

**删除**：
- `knightHP`（骑士专用双格HP） — 统一为通用 `hp`

**新增**：
- `hp` — 当前生命值，创建时从 `UnitStats[fType].hp` 初始化
- `maxHp` — 最大生命值
- `attackTimer` — 攻击冷却计时器，初始为 0，每帧递减，归零时可发动攻击

### 1.4 近战战斗流程

替代现有 `Combat.processCombat()` 中的布尔生死判定：

1. 攻击方进入碰撞距离（保持现有 `(raA + raB) * 0.8` 判定）
2. 检查 `attackTimer <= 0`，若是：
   - 计算伤害 = `ATK × DamageMultiplier[攻击方fType][防御方fType]`
   - 血月期间 `× 1.5`
   - 防御方 `hp -= damage`
   - 重置 `attackTimer = atkInterval`
   - 生成伤害数字粒子
3. 双方同时判定（A打B的同时B也可以打A）
4. `hp <= 0` 的单位标记 `alive = false`
5. 存活方重置 `state = "following"` 寻找下一个目标

**关键区别**：不再"碰一下就判生死"，而是停在原地互砍。

### 1.5 远程战斗（弓手）

弓手行为保持不变（射箭），但箭矢命中后的结算改为数值制：

- 箭矢命中敌方单位：`damage = ArcherATK × DamageMultiplier.archer[目标fType]`
- 箭矢命中领主：`damage = ArcherATK`（20点，不乘克制倍率）
- 箭矢命中Boss：`damage = ArcherATK`（20点）
- 不再使用 `COMBAT_TABLE.archer[fType]` 的布尔判定

### 1.6 单位对领主伤害

现有的"神风特攻"（碰领主自杀造成伤害）改为**持续输出**：

- 单位进入领主碰撞距离后，按 `attackTimer` 周期性造成伤害
- 伤害 = 单位 ATK 值（士兵25、骑士35、弓手20通过箭矢）
- **单位不再自杀**，持续站在领主旁边输出
- 领主的 `invincibleTimer`（0.3s无敌帧）改为只防止同一帧多次结算，不影响持续战斗

### 1.7 单位对Boss伤害

同上，改为持续输出：
- 单位进入Boss碰撞距离后按 `attackTimer` 周期性造成伤害
- 伤害 = 单位 ATK 值
- 单位不再自杀

## 二、领主即移动据点

### 2.1 架构变化

**删除**：
- `Stronghold.lua` 中的独立据点实体创建（`createStronghold`）
- `GS.strongholds` 数组
- 据点渲染代码（Renderer 中画据点建筑的部分）
- 防御塔崩溃时间机制（`StrongholdCollapseTime`）— 领主自带防御塔不崩溃

**保留并迁移到领主上**：
- 防御塔攻击逻辑（从据点上搬到领主上）
- 复活机制（从据点坐标改为领主死亡位置）

### 2.2 领主新增字段

```lua
lord.strongholdHP      = 300   -- 据点血量（复活资本）
lord.strongholdMaxHP   = 300   -- 据点最大血量
lord.towerActive       = true  -- 防御塔是否启用
lord.towerTimer        = 0     -- 防御塔冷却计时
lord.lordMode          = "attack"  -- "attack" | "defend"
lord.deathX            = nil   -- 死亡位置X（用于复活）
lord.deathY            = nil   -- 死亡位置Y
```

### 2.3 双血条机制

| 血条 | 初始值 | 扣血条件 | 归零后果 |
|------|--------|---------|---------|
| 领主HP | 100 | 被敌方单位/Boss攻击 | 暂时死亡，进入复活流程 |
| 据点HP | 300 | 每次领主死亡时扣100（血月扣150） | 永久淘汰，无法复活 |

据点HP的消耗规则：
- 领主HP归零触发死亡 → 据点HP **-100**
- 血月期间领主死亡 → 据点HP **-150**（×1.5）
- 领主活着时据点HP**不被消耗**
- 据点HP不可恢复

### 2.4 复活机制

1. 领主 HP 归零
2. 记录死亡位置 `(deathX, deathY)`
3. 据点HP -= 100（血月150）
4. 检查据点HP：
   - `> 0`：进入复活倒计时（3秒），倒计时结束后在死亡位置复活
   - `<= 0`：永久淘汰
5. 复活时：HP恢复50%，3秒无敌，带2农民+1士兵
6. 惩罚不变：损失50%资源，掉落战利品箱

### 2.5 防御塔（跟随领主）

从据点迁移到领主身上，逻辑基本不变：
- 每 1.5 秒攻击范围内最近的敌方单位
- 射程 150 像素
- 对单位造成 15 点伤害（不再一击杀，因为单位现在有HP了）
- 对敌方领主造成 15 点伤害
- **不再有5分钟崩溃机制**（领主即据点，塔一直跟着）

### 2.6 胜利条件

- 原来：所有敌方据点被摧毁 + 无复活中领主
- 现在：所有敌方领主的据点HP归零 + 领主本体也死亡

## 三、领主双模式：攻击 / 防守

### 3.1 替代现有战术指令

**删除**：三档战术指令（default / defend / charge）
**新增**：两档模式（attack / defend）

切换方式：Q 键切换，删除 E 键冲锋指令。

### 3.2 攻击模式（默认）

| 属性 | 值 |
|------|-----|
| 领主移速倍率 | 1.0 |
| 士兵搜索半径倍率 | 1.0 |
| 士兵行为 | 主动追击范围内敌人 |
| 领域光环半径 | 220像素（正常） |
| 领域伤害 | 无 |
| 防御塔 | 正常（单体15伤害/1.5秒） |

### 3.3 防守模式

| 属性 | 值 |
|------|-----|
| 领主移速倍率 | 0.7 |
| 领域光环半径 | 330像素（1.5倍） |
| 士兵行为 | 不主动追击，仅拦截进入领域的敌人 |
| **领域持续伤害** | 每0.5秒对领域内所有敌方单位造成5点伤害（等效10点/秒） |
| **领域持续伤害（对敌方领主）** | 每0.5秒造成2.5点伤害（等效5点/秒） |
| **领域减速** | 领域内敌方单位移速降低30% |
| 防御塔 | 正常（单体15伤害/1.5秒，叠加在领域伤害之上） |

Config 新增字段：
```lua
LordModes = {
    attack = {
        speedMul = 1.0,
        auraMul = 1.0,
        searchMul = 1.0,
        auraDamage = 0,
        auraSlowPct = 0,
    },
    defend = {
        speedMul = 0.7,
        auraMul = 1.5,
        searchMul = 0.4,
        auraDamage = 10,       -- 每秒对单位伤害
        auraDamageToLord = 5,  -- 每秒对敌方领主伤害
        auraSlowPct = 0.3,     -- 减速30%
        auraDmgInterval = 0.5, -- 结算间隔
    },
},
```

### 3.4 领域伤害结算

在 `Stronghold.updateStrongholds()`（将重命名为领主更新逻辑）中新增：

1. 检查领主 `lordMode == "defend"`
2. 遍历所有敌方单位，计算与领主距离
3. 距离 < `auraRadius`（330像素）的敌方单位：
   - 每0.5秒扣5点HP（血月×1.5）
   - 移速 ×0.7
4. 距离 < `auraRadius` 的敌方领主：
   - 每0.5秒扣2.5点HP（血月×1.5）
5. 生成伤害数字粒子（频率降低，每1秒显示一次合并伤害数字，避免刷屏）

### 3.5 AI领主模式选择

AI领主根据兵力和态势自动切换模式：
- 战斗单位数量 < 3 或 领主HP < 30% → 自动切换防守模式
- 战斗单位数量 >= 5 且 领主HP > 60% → 自动切换攻击模式
- 其他情况保持当前模式

## 四、视觉表现

### 4.1 不显示据点建筑

- 地图上**没有任何据点建筑图形**
- 据点概念仅通过领主头顶的第二条血条体现
- 删除 Renderer 中所有据点建筑绘制代码

### 4.2 领主头顶双血条

| 血条 | 位置 | 颜色 | 显示对象 |
|------|------|------|---------|
| 据点HP | 上方 | 橙/金色 | 仅玩家领主显示 |
| 领主HP | 下方 | 绿→红渐变 | 所有领主显示 |

- AI领主只显示领主HP血条，不显示据点HP
- 血条宽度 = 领主半径 × 2

### 4.3 领域光环视觉

| 模式 | 光环颜色 | 光环大小 |
|------|---------|---------|
| 攻击模式 | 阵营色半透明（保持现有） | 220像素 |
| 防守模式 | 偏紫/红色脉冲 + 阵营色 | 330像素 |

防守模式光环增加脉冲动画（alpha 在 0.1~0.3 之间周期变化），表示正在造成持续伤害。

### 4.4 模式切换UI提示

切换模式时在领主头顶短暂显示文字：
- 切换到攻击模式："进攻！"（红色，1秒消失）
- 切换到防守模式："防守！"（蓝色，1秒消失）
